Fix: Use Railway’s database, not localhost

Your app is running on Railway but DATABASE_URL points to localhost. On the server there is no Postgres on localhost, so the connection fails.

Do this:

  1. In the Railway dashboard, open your project.
  2. Add a PostgreSQL plugin if you don’t have one (New → Database → PostgreSQL).
  3. Open your app service (the one that runs this code). Go to Variables.
  4. Remove any manual DATABASE_URL or POSTGRES_* variables you added. Railway will inject the correct DATABASE_URL when the database is linked.
  5. In your app service, go to Settings → Connect (or the “Variables” tab and use “Add a reference”) and link the PostgreSQL service so that DATABASE_URL is provided automatically.
  6. Redeploy the app. The debug page should then show a non-localhost URL and a successful connection.
