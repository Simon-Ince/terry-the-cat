Fix: Use Railway’s database, not localhost

Your app is running on Railway but DATABASE_URL points to localhost. On the server there is no Postgres on localhost, so the connection fails.

Do this:

  1. Open the PostgreSQL service in your project → Connect tab.
  2. Copy the “Postgres Connection URL” (the full connection string).
  3. Open your app service → Variables. Remove any existing DATABASE_URL that uses localhost.
  4. Add DATABASE_URL and paste the URL you copied (it should look like postgresql://...@...railway.app:PORT/railway or use postgres.railway.internal).
  5. Redeploy. The app also checks DATABASE_PRIVATE_URL and DATABASE_PUBLIC_URL if those are set by linking.
